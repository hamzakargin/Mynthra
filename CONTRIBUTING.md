# Contributing to Mynthra Theme

Thank you for your interest in contributing!  
Below are simple guidelines to help keep the project clean, consistent, and easy to maintain.

---

## üõ† How to Contribute

### 1. Fork the Repository
Click the **Fork** button to create your own copy of the project.

### 2. Create a New Branch
Use a clear and descriptive branch name:
- fix/color-contrast-issue
- feature/new-highlight-style
- docs/update-readme

### 3. Make Your Changes
Please follow the existing folder structure and naming conventions.  
Avoid unnecessary or unrelated edits.

### 4. Test the Theme
Before opening a PR:
- Install the extension locally  
- Test syntax highlighting  
- Check contrast, readability, and overall consistency  
- Verify dark/light mode behavior (if applicable)

---

## ‚úî Pull Request Guidelines

- Provide a short and clear description of your changes  
- Include screenshots if you made visual or color modifications  
- Keep commits clean, meaningful, and organized  
- One feature/fix per PR (please avoid mixed changes)  
- Make sure your code/theme changes do not introduce regressions

---

## üêû Reporting Issues
If you found a problem, feel free to open an **Issue**.  
Please include the following to help us understand the problem:

- VS Code version  
- Mynthra Theme version  
- Screenshots (optional but helpful)  
- A short description of the issue  
- Steps to reproduce (if applicable)
